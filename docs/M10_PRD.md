## 1 · 目标 (Goals)

**让 paper_rag 不再止步于"答案文本"，而是产出研究者真正能用的交付物**：

1. **Markdown 综述生成器**：多 paper → 分章 + 引用 + 参考文献的完整草稿
2. **PPT 适配**：综述 → reading group / 学术报告演讲卡片
3. **Word 适配**：综述 → 可直接交导师的 .docx
4. **LaTeX + .bib 一体化**：related work 段落 + .bib 文件一键打包
5. **HTTP + LangChain Tool 双形态**：与 M8 设计一致

---

## 2 · 非目标 (Non-Goals)

- ❌ 真正的"自动写论文"（ML 写作辅助是另一个项目）
- ❌ 自动生成图表 / 实验结果可视化（M12 多模态时再做）
- ❌ 协同编辑 / 版本管理（用 Git + Notion 处理）
- ❌ 任意格式（PDF 留待用户自己 docx → pdf）

## 3 · 现状 (Context)

paper_rag M0-M8 已具备：
- `qa_agentic.answer()` 返回 answer + cites + chunks
- `bibtex_export.py` 已有（M6 #31）

## 4 · 用户故事 (User Stories)

### US-1：综述初稿（最高频）
> 张同学：上传 10 篇 RAG 论文，发起命令 "生成 RAG 综述"。  
> 系统：30 秒后输出一份 5000 字 Markdown，含 Introduction / Methods Comparison / Open Problems / References，每个事实都有 `[chunk:xxx]` 引用，最后是去重的参考文献列表。

### US-2：reading group PPT
> 王同学：选 1 篇论文 → "做成 30 分钟讲解 PPT"  
> 系统：输出 .pptx，含 12 张：Title / Background / Method / Algorithm Block / Experiments / Ablation / Pros / Cons / Open Questions / References / 备注页（讲稿）

### US-3：导师交差 Word
> 李同学：写完 Markdown 综述 → "导出 Word"  
> 系统：转 .docx，保留章节层级、引用上标、参考文献字段，**配色与字体走标准学术模板**。

### US-4：LaTeX bib 包
> 赵同学：写论文到 related work → "生成这 8 篇的 bib + 一段 related work"  
> 系统：输出 zip：references.bib + related_work.tex（用 \cite{} 标准引用），可直接 \input。

## 5 · 接口设计 (API)

### 5.1 New Router endpoint

```http
POST /api/paper_rag/deliver
Content-Type: application/json
Authorization: <BetterAuth cookie>

{
  "format": "markdown_survey" | "pptx" | "docx" | "latex_bib",
  "paper_ids": ["arxiv:2310.11511", "arxiv:2305.06983"],
  "title": "Self-Reflective RAG: A Survey",  // optional
  "options": {                                  // format-specific
    "max_words": 5000,                          // markdown_survey
    "n_slides": 12,                             // pptx
    "tone": "academic" | "casual",              // pptx/docx
    "tex_style": "natbib" | "biblatex"          // latex_bib
  }
}
```

响应（小文件直接返 base64，大文件返路径）：

```json
{
  "format": "markdown_survey",
  "filename": "rag_survey_2026-05-20.md",
  "content_base64": "...",       // 文件内容
  "content_type": "text/markdown",
  "size_bytes": 17234,
  "metadata": {
    "n_papers": 2,
    "n_citations": 47,
    "abstain_decisions": [...],   // 透明：哪些 paper retrieve 失败
    "trace_id": "..."
  }
}
```

### 5.2 LangChain Tool

```python
@tool("paper_deliver", parse_docstring=True)
def paper_deliver_tool(
    format: str,                 # one of: markdown_survey | pptx | docx | latex_bib
    paper_ids: str,              # comma-separated
    title: str = "",
    options_json: str = "{}",
) -> str:
    """Generate a deliverable artifact from indexed papers."""
```

## 6 · 数据流 (Pipeline)

```
                       ┌─ markdown_survey ─→ deliver/survey_md.py
                       │
POST /deliver ──→ deliver_dispatch ─→ pptx ──→ deliver/pptx.py (uses python-pptx)
                       │
                       ├─ docx ─────────→ deliver/docx.py (uses python-docx)
                       │
                       └─ latex_bib ────→ deliver/latex_bib.py (extends bibtex_export)

每个生成器内部 pipeline:
  paper_ids → bulk paper_qa() (per-paper) → planner LLM call → 
  template rendering → bytes → response
```

### 6.1 综述生成器（最复杂）

```python
def generate_survey(paper_ids, title, max_words=5000):
    # Step 1: per-paper deep-dive
    summaries = []
    for pid in paper_ids:
        out = qa_agentic.answer(
            f"Summarize the paper {pid} in 200 words: motivation, method, results, limitations.",
            paper_ids=[pid],
        )
        summaries.append({"pid": pid, "summary": out["answer"], "cites": out["citations"]})
    
    # Step 2: cross-paper synthesis (single LLM call with all summaries)
    sections = synthesize_outline(summaries, max_words=max_words)
    # sections = {"intro": "...", "methods_comparison": "...", "open_problems": "..."}
    
    # Step 3: render Markdown
    return render_markdown_template(title, sections, all_cites=...)
```

## 7 · 模板与样式

### 7.1 Markdown 综述模板（最小化）

```markdown
# {title}

> Generated by paper_rag on {date}. {n_papers} papers, {n_citations} citations.

## Introduction
{intro_section}

## Method Comparison
{methods_section}

## Open Problems
{open_problems_section}

## References
1. [chunk:xxx] — Paper Title (Author, Year). arxiv:XXXX
...
```

### 7.2 PPT 模板

12 张固定卡片：Title / Outline / Background / Method × 3 / Experiments / Ablation / Pros & Cons / Open Questions / Q&A / References  
基于 `python-pptx` `Presentation()` 默认布局，**不引入第三方主题**（第一版尽量简单）。

### 7.3 LaTeX bib 包

```
output.zip
├── references.bib           # 复用 bibtex_export
└── related_work.tex         # 一段 \cite{} 引用的 paragraph
```

## 8 · 复用 vs 自建

| 能力 | 决策 |
|---|---|
| Markdown 渲染 | **自建**（一个 jinja2 模板 + paper_rag.qa_agentic 调用） |
| python-pptx | **第三方库**（pyproject 加进 `deliver` extra） |
| python-docx | **第三方库**（同上） |
| .bib 生成 | **复用** `paper_rag.tools.bibtex_export`（M6 #31 已有） |
| LLM 综述合成 | **复用** `qa_agentic.answer` + 一次额外 cross-paper synthesis |

## 10 · 风险与缓解

| 风险 | 缓解 |
|---|---|
| python-pptx / python-docx 在 docker 容器里的字体问题 | 用默认主题，不依赖系统字体 |
| 大综述 LLM 输出 > 8000 token 被截断 | 分章生成（每章独立 LLM call），最后 join |
| 跨 paper synthesis 语义飘 | synthesis prompt 里强制只用前面 summaries 里的事实 + 必带 cite |
| zip 结构损坏 | `zipfile.ZIP_DEFLATED` + 单测验证可解压 |
